## 1. 设计系统

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **信息优先** | 数据驱动，减少装饰性元素，让内容和状态清晰可见 |
| **一致节奏** | 严格遵循 4px 网格体系，所有间距 / 内边距 / 圆角均为 4 的倍数 |
| **暗色优先** | 默认暗色模式，亮色同步适配，不可出现仅在暗色下可读的界面 |
| **中文优先** | 所有用户可见文本为中文，中文字体栈优先 |
| **渐进复杂** | 简单任务少点击，复杂操作分步骤引导 |

### 1.6 阴影

| Token | 亮色 | 暗色 | 用途 |
|-------|------|------|------|
| `shadow-sm` | `0 1px 2px 0 rgb(0 0 0 / 0.05)` | `0 1px 2px 0 rgb(0 0 0 / 0.3)` | 卡片默认 |
| `shadow` | `0 1px 3px 0 rgb(0 0 0 / 0.1)` | `0 1px 3px 0 rgb(0 0 0 / 0.4)` | 下拉菜单 |
| `shadow-md` | `0 4px 6px -1px rgb(0 0 0 / 0.1)` | `0 4px 6px -1px rgb(0 0 0 / 0.4)` | 对话框 |
| `shadow-lg` | `0 10px 15px -3px rgb(0 0 0 / 0.1)` | `0 10px 15px -3px rgb(0 0 0 / 0.5)` | Modal |
| `shadow-xl` | `0 20px 25px -5px rgb(0 0 0 / 0.1)` | `0 20px 25px -5px rgb(0 0 0 / 0.6)` | Drawer |

### 1.7 暗色模式

所有组件必须同时提供亮/暗两套样式。规则：

- 背景: `bg-white` → `dark:bg-gray-900`（页面）/ `dark:bg-gray-800`（卡片）
- 文字: `text-gray-900` → `dark:text-gray-100`
- 辅助文字: `text-gray-500` → `dark:text-gray-400`
- 边框: `border-gray-200` → `dark:border-gray-700`
- 输入框背景: `bg-white` → `dark:bg-gray-800`
- 阴影: 暗色下不透明度翻倍

### 1.8 过渡动画

```css
--transition-fast: 150ms ease;
--transition-normal: 200ms ease;
--transition-slow: 300ms ease;
```

所有可交互元素需有 `transition`；面板切换使用 fade + slide；列表增删使用列表动画。

## 2. 布局架构

### 2.1 整体布局

```
┌──────────────────────────────────────────────────┐
│  Sidebar (64px)  │       Main Area              │
│                  │  ┌───────────────────────┐   │
│  ┌──────────┐   │  │  Topbar              │   │
│  │  Logo    │   │  │  breadcrumb + user    │   │
│  └──────────┘   │  └───────────────────────┘   │
│                  │                               │
│  ┌──────────┐   │  ┌───────────────────────┐   │
│  │ Nav         │   │  Page Content          │   │
│  │ Items       │   │  (scrollable)          │   │
│  │             │   │                        │   │
│  │             │   │                        │   │
│  └──────────┘   │  └───────────────────────┘   │
│                  │                               │
│  ┌──────────┐   │  ┌───────────────────────┐   │
│  │ 底部操作   │   │  │  Optional: Detail    │   │
│  │ (折叠/主题) │   │  │  Panel (Drawer)      │   │
│  └──────────┘   │  └───────────────────────┘   │
└──────────────────────────────────────────────────┘
```

- **Sidebar**: 固定宽度 240px (`w-60`)，深色背景 (`bg-gray-900`)，可折叠为仅图标模式 (64px)
- **Topbar**: 高度 56px (`h-14`)，包含面包屑导航、搜索快捷键、用户菜单
- **Content**: 弹性撑满，内部滚动 (`overflow-y-auto`)
- **Detail Panel**（可选）：右侧滑出 Drawer，宽度 480px，用于查看/编辑详情

### 2.2 响应式断点

| Breakpoint | Width | 布局变化 |
|-----------|-------|---------|
| `sm` | ≥640px | 单列 → 双列 |
| `md` | ≥768px | Sidebar 文本显示 |
| `lg` | ≥1024px | 三列网格 |
| `xl` | ≥1280px | 全宽布局 |
| `2xl` | ≥1536px | 内容容器 max-w 限制 |

### 2.3 布局变体

| 变体 | 触发条件 | 说明 |
|------|---------|------|
| **登录页** | 未认证 | 居中卡片，无 Sidebar/Topbar |
| **标准页** | 大多数页面 | Sidebar + Topbar + Content |
| **工作流编辑器** | `/workflows/:id/edit` | 全屏布局，隐藏 Sidebar，Topbar 变浮动工具条 |
| **全屏** | 执行监控 | 隐藏 Sidebar + Topbar，仅内容区 |

## 3. 路由设计

### 3.1 React Router 结构

```
/login                    → LoginPage              [公开]
/register                 → LoginPage (register)    [公开]

/                         → DashboardPage           [受保护]
/agents                   → AgentListPage
/agents/:id               → AgentDetailPage
/agents/:id/edit          → AgentEditPage
/agents/new               → AgentCreatePage

/tools                    → ToolListPage
/tools/new                → ToolCreatePage

/workflows                → WorkflowListPage
/workflows/:id            → WorkflowDetailPage
/workflows/:id/edit       → WorkflowEditorPage  (全屏)
/workflows/new            → WorkflowCreatePage

/executions               → ExecutionListPage
/executions/:id           → ExecutionDetailPage (含实时流)

/mcp                      → MCPListPage
/mcp/servers/:id          → MCPServerDetailPage

/memories/:agentId        → MemoryListPage

/settings                 → SettingsPage
```

### 3.2 路由守卫

```
<Routes>
  <Route path="/login" element={<PublicRoute><LoginPage /></PublicRoute>} />
  <Route element={<AuthLayout />}>     ← 统一提供 Sidebar + Topbar
    <Route path="/" element={<Navigate to="/dashboard" />} />
    <Route path="/dashboard" element={<DashboardPage />} />
    <Route path="/agents" element={<AgentListPage />} />
    <Route path="/agents/:id" element={<AgentDetailPage />} />
    ...
  </Route>
</Routes>
```

- `PublicRoute`: 已登录则重定向到 `/`
- `AuthLayout`: 未登录则重定向到 `/login`
- 所有未匹配路由 → 404 页面

## 6. 交互流程

### 6.1 关键用户流程

#### Flow A: 创建 Agent

```
登录 → Agent 列表 → 点击 "+创建" → 填写表单
  → 保存成功 → 跳转 Agent 详情 → 可用
```

#### Flow B: 构建工作流

```
工作流列表 → 点击 "+创建" → 进入全屏编辑器
  → 从左侧拖拽节点 → 连接节点 → 配置节点属性
  → 点击 "验证" → 通过 → 点击 "保存"
  → 点击 "执行" → 跳转执行详情 → 实时查看进度
```

#### Flow C: 监控执行

```
执行列表 → 点击运行中的执行 → 执行详情页
  → SSE 实时推送步骤完成 → 节点动画推进
  → 完成/失败 → 查看日志 + 输入输出
  → 可点击 "重新执行"
```

#### Flow D: 注册 MCP 工具

```
MCP 服务器列表 → "+添加服务器" → 填写服务器信息
  → 保存 → 自动拉取工具列表 → 选中工具导入
  → 工具出现在"工具管理"列表中 → 可在工作流中使用
```

### 6.2 全局交互约定

| 交互 | 行为 |
|------|------|
| **右键菜单** | 表格行右键显示操作菜单 |
| **键盘快捷键** | `Ctrl+K` 全局搜索 / `Escape` 关闭弹窗 / `Ctrl+S` 保存 |
| **Toast** | 操作成功/失败统一使用全局 Toast，3s 自动消失 |
| **确认对话框** | 所有删除操作须二次确认 |
| **表单校验** | 字段失焦时实时校验，提交时全量校验 |
| **无限滚动** | 日志等长列表使用虚拟滚动 |
| **Debounce** | 搜索输入 300ms debounce 后发起请求 |
